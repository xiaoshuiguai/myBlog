承建单位确认：
信息技术中心与学校项目建设部门协同合作，发起学校信息化建设项目需求。依据业务建设内容，确定合作承建单位。
承建单位需明确承担方项目团队成员及其分工情况。
需求调研：
信息技术中心组织承建单位开展建设需求调研，承建单位的调研结果内容（无论是现场调研情况还是后续形成的文档）均需向建设部门进行需求内容确认。
初版建设方案与可行性论证：
承建单位根据调研结果制定初步方案建设。
信息技术中心组织业务建设部门与专家组，对方案的需求合理性、建设集约性、方案可行性、预算合理性以及是否符合学校信息化建设总体要求和标准规范等情况进行论证。
方案确认与项目排期：
承建单位依据论证意见，形成最终建设方案。
将最终建设方案提交至信息技术中心领导小组，由其牵头各部门进行审议。审议通过后，进行项目排期。
开发阶段：
信息技术中心负责对信息化项目的实施进行监督和全过程管理，在开发阶段主要监督项目进程及质量。
项目承建单位应定期向信息技术中心及建设部门报送项目进展情况（包括里程碑事件）。
信息化项目需按照项目建设方案实施。若项目建设内容或技术方案确需调整，项目建设部门或承建单位应及时提出调整申请，由信息技术中心综合评估后提出调整意见。当项目建设内容调整超过 10% 时（可根据项目内容进行沟通调整），需报信息技术中心领导小组审批决策。
信息化项目应当按照项目建设排期计划实施。若因故不能按期完成项目计划，项目承建单位应至少提前 3 个工作日（可按项目周期进行沟通调整）向信息中心领导小组报备。若项目出现严重逾期或重大投资损失等问题，应按有关规定进行整改或暂停项目建设。
测试阶段：
承建单位完成开发后，需确保通过项目功能及性能测试，并向信息技术中心上交源代码安全审计报告及测评报告。
项目测试通过后，申请项目初验，由项目建设部门负责组织项目初验。
对于测试过程中项目建设内容确需调整的，项目建设部门应及时提出调整申请，由信息技术中心综合评估后提出调整意见。当项目建设内容调整超过 10% 时（可按项目内容进行沟通调整），需报信息技术中心领导小组审批决策。
上线或变更：
正式环境上线或者变更时，承建单位应至少提前 3 个工作日通知信息技术中心及项目建设部门，并向信息技术中心提交《上线变更单》。
信息技术中心因上线或者变更操作对实际生产使用产生影响时，需提前进行全校通知。
试运行：
初验通过的项目进入试运行阶段，承建单位须协调建设部门，对试运行过程中出现的各类问题进行详细记录，并提供试运行报告。
对于初验过程中项目建设内容确需调整的，项目建设部门应及时提出调整申请，由信息技术中心综合评估后提出调整意见。当项目建设内容调整超过 10% 时（可按项目内容进行沟通调整），需报信息技术中心领导小组审批决策。
承建单位需解决试运行期间发现的问题，直至项目稳定运行。
验收：
正式版本推广上线后，经过至少一个完整功能使用周期稳定试运行的项目，完成项目最终验收。
运维：
建设部门应收回承建单位所有系统权限。
建设部门需安排本部门专员负责业务系统的日常运行维护和管理，包括业务层面的维护和管理、系统使用和推广、系统数据安全管理等。
信息技术中心进行系统层技术支持与运维支持。
承建单位需根据学校需求进行系统服务支持。
